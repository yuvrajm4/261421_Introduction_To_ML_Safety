# Exercise 1 — ML Safety Basics

## Overview

This exercise introduces the basic ideas behind Machine Learning Safety. The goal is to understand why ML systems can fail, why testing them is different from testing normal software, and how risk-based thinking is used when ML models are deployed in real-world systems.

## Main Topics

- Difference between traditional software and ML-based systems
- Basic ML safety terminology
- Risk levels for AI systems
- Examples of unsafe or unreliable ML behaviour
- Importance of validation, testing, and monitoring

## Key Learning Points

ML models do not follow manually written rules only. They learn behaviour from data, which means their performance depends strongly on the training data, test data, and deployment environment.

A model can work well on average but still fail in rare or safety-critical situations. Because of this, safety evaluation must look beyond accuracy and include failure cases, edge cases, and risks.
